# ParkSense - Quick Start Guide

## 🚀 Launch the Demo (3 Steps)

### Step 1: Start the Application
```bash
npm run dev
```
Open your browser to: `http://localhost:5173`

### Step 2: Configure for Best Demo
1. Set **Demo Speed** to **10x** (upper control bar)
2. Select **"Evening Peak"** scenario for maximum impact
3. Press **F11** for fullscreen mode

### Step 3: Enter Presentation Mode
Press **P** key to hide controls and enter clean presentation view

## 🎯 60-Second Demo Flow

1. **Press 1 (Dashboard)** - 15 seconds
   - Point to real-time map with color-coded zones
   - Show ANPR feed scrolling with vehicle detections
   - Highlight live revenue counter increasing

2. **Press 2 (Mobile App)** - 15 seconds
   - Click "Find Now" → Select a zone → Show QR code reservation
   - Navigate through: Search → Reserve → Payment flow

3. **Press 3 (Analytics)** - 15 seconds
   - Show 24-hour occupancy trends
   - Point to revenue by zone chart
   - Emphasize impact metrics: **2,450 searches eliminated, 1,200 hours saved, 450 kg CO₂ reduced**

4. **Press 4 (Pricing)** - 10 seconds
   - Move the occupancy slider from 20% to 95%
   - Watch price change from ₹16 (discount) to ₹100 (surge)
   - Explain AI factors: occupancy, time, events, weather

5. **Press 5 (VMS)** - 5 seconds
   - Show LED boards with real-time parking guidance
   - Mention integration with traffic signals and Google Maps

## 💡 Quick Tips

### Before Presentation
- ✅ Test all 5 views (press keys 1-5)
- ✅ Ensure demo speed is set to 10x for impressive action
- ✅ Select impactful scenario (Evening Peak or Event Day)
- ✅ Clear browser cache for optimal performance
- ✅ Have backup browser open (Chrome + Firefox)

### During Presentation
- Start with Dashboard view (most impressive)
- Let simulation run for 5-10 seconds to show real-time updates
- Use keyboard shortcuts for smooth transitions
- If judges ask questions, know which view shows the answer:
  - **Technical details?** → Press 4 (AI Pricing)
  - **User experience?** → Press 2 (Mobile App)
  - **Business metrics?** → Press 3 (Analytics)
  - **Real-world deployment?** → Press 5 (VMS Integration)

### After Presentation
- Press **R** to reset and show reliability
- Switch to **Event Day** scenario for extreme stress test
- Demonstrate **what-if simulator** in Pricing view

## 🐛 Troubleshooting

### If simulation seems slow
- Increase demo speed to 5x or 10x
- Check browser dev tools for any errors (F12)

### If colors look off
- Ensure browser zoom is at 100%
- Try different browser (Chrome recommended)

### If animations are choppy
- Close other tabs/applications
- Disable browser extensions temporarily
- Ensure hardware acceleration is enabled
